\doxysubsection{详细描述}
Abstract class that defines an interface for computing and comparing equivalence classes 

Equivalence relations are utilized in order to test if two trajectories are belonging to the same equivalence class w.\+r.\+t. the current obstacle configurations. A common equivalence relation is the concept of homotopy classes. All trajectories belonging to the same homotopy class can CONTINUOUSLY be deformed into each other without intersecting any obstacle. Hence they likely share the same local minimum after invoking (local) trajectory optimization. A weaker equivalence relation is defined by the concept of homology classes (e.\+g. refer to \mbox{\hyperlink{classteb__local__planner_1_1HSignature}{HSignature}}).

Each \mbox{\hyperlink{classteb__local__planner_1_1EquivalenceClass}{Equivalence\+Class}} object (or subclass) stores a candidate value which might be compared to another \mbox{\hyperlink{classteb__local__planner_1_1EquivalenceClass}{Equivalence\+Class}} object.

\begin{DoxyRemark}{备注}
Currently, the compute\+Equivalence\+Class method is not available in the generic interface \mbox{\hyperlink{classteb__local__planner_1_1EquivalenceClass}{Equivalence\+Class}}. Call the \char`\"{}compute\char`\"{}-\/methods directly on the subclass. 
\end{DoxyRemark}
